第一个Mybatis

1、简介

1.1、什么是 MyBatis？

MyBatis 是一款优秀的持久层框架，它支持自定义 SQL、存储过程以及高级映射。MyBatis 免除了几乎所有的 JDBC 代码以及设置参数和获取结果集的工作。MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

1.2、特点

  • 简单易学：本身就很小且简单。没有任何第三方依赖，最简单安装只要两个jar文件+配置几个sql映射文件易于学习，易于使用，通过文档和源代码，可以比较完全的掌握它的设计思路和实现。
  • 灵活：mybatis不会对应用程序或者数据库的现有设计强加任何影响。 sql写在xml里，便于统一管理和优化。通过sql语句可以满足操作数据库的所有需求。
  • 解除sql与程序代码的耦合：通过提供DAO层，将业务逻辑和数据访问逻辑分离，使系统的设计更清晰，更易维护，更易单元测试。sql和代码的分离，提高了可维护性。
  • 提供映射标签，支持对象与数据库的orm字段关系映射
  • 提供对象关系映射标签，支持对象关系组建维护
  • 提供xml标签，支持编写动态sql。

1.3、如何获得mybatis

  • maven仓库
  • Github：https://github.com/mybatis/mybatis-3/releases
  • 中文文档：https://mybatis.org/mybatis-3/zh/index.html

应用层协议原理

应用程序体系结构

  • C/S结构(client-server architecture)

服务器处于长期打开的状态，服务于来自其他客户的主机的请求。服务器有永久性域名/地址(IP)使得客户端能够访问。值得注意的是，客户端于客户端之间没法进行通信。客户机与服务器进行通信，间歇性接入网络，可能使用动态的IP地址，不会与其他客户机直接通信。比如Web服务就是典型的C/S结构。

  • P2P结构(Peer-Peer architecture)

无永远在线的服务器，主机之间均为对等方，节点可以改变IP，间接性接入网络。比如文件共享是典型的P2P结构。该结构优点是自扩展性(self-scalability)，缺点是难以管理。

  • 混合结构(Hybrid architecture)

比如Napster是混合结构，文件传输采用P2P结构，文件搜索采用集中式的C/S结构。每个节点向中央服务器登记自己的内容，每个节点向中央服务器提交查询请求，查找感兴趣的内容。

我尝试暴力无果后，想到了前缀和后缀和能解决该问题。该题需要两个数组分别维护前缀和后缀和。

  • pre数组维护前i个学生flag==0的数量
  • suf数组维护从第i个学生到最后一个学生flag==1的数量

这样我们要的结果就等价于pre[i-1]+suf[i]，我们用ans存储最大的预测正确次数，t存储预测的阈值就可以完成该道题。

P.S. 这里为了偷懒，直接用了头文件的sort()函数，第三个参数cmp是比较两个数据的优先规则，这里是y越小排越前面.

P.S.S 为了简化运算，这里用了if(a[i].y!=a[i-1].y)，只有前后的y不相同的时候，我们才开始计算，这样每次的a[i].y都是y相同中i最小的那一个进行计算，避免重复计算。

P.S.S.S 为了解决【多个不同t都能达到最大的预测正确次数】的问题，这里用了if(pre[i-1]+suf[i]>=ans)如果=成立，正好是【多个不同t都能达到最大的预测正确次数】的问题，我们取较大的t，所以t=max(ti)。又因t又是a[i].y赋值过去的，a[i].y是排序好了的，所以我们直接t=a[i].y，达到了取等时，用较大t的目的。

结果分析

时间复杂度为$O(m)$,m是第一行参数——人数m。

空间复杂度为$O(3N)=O(N)$,N是我们数组开辟空间大小，题目说明100%数据的m都比10^5小，所以也为O(m_max)。

面积$s=min(h[i],h[j])*(j-i)$

双指针做法：

我们让$i,j$俩指针分别指向数组的开头与结尾，那么接下来比较两个指针指向的数字。

如果我们移动数字较大的那个指针，那么「两个指针指向的数字中较小值」$(j-i)$会减小，而「指针之间的距离」$min(h[i],h[j])$会减小，那么这个乘积$s=min(h[i],h[j])*(j-i)$会减小。因此，我们移动数字较大的那个指针是不合理的。因此，我们移动 数字较小的那个指针。

所以结论是我们移动数字较小的那个指针。

522.学生出勤记录

题目描述

原题链接地址

分析

很明显的动态规划题。

三种状态方程：

若第$i$天的出勤状态是$A$，则状态转移方程为

$dp[i][1][0]=dp[i][1][0]+\displaystyle \sum^{2}_{k=0}dp[i−1][0][k]$

若第$i$天的出勤状态是$L$，则状态转移方程为

$dp[i][j][0]=dp[i][j][0]+dp[i−1][j][k-1]$

若第$i$天的出勤状态是$P$，则状态转移方程为

$dp[i][j][0]=dp[i][j][0]+\displaystyle \sum^{2}_{k=0}dp[i−1][j][k]$

由于$i$是从第一天一直遍历到最后一天的，所以我们可以用$newdp[j][k]$来存储当天的所有状态情况，用memset(newdp,0,sizeof(newdp))来初始化该数组，然后在一天结束之后，用memcpy(dp, newdp, sizeof(dp))将$newdp$的内容拷贝到$dp$中去，从而达到省下$i$的空间。

时间复杂度为$O(n)$，因为$i$遍历了$n$天。

空间复杂度为$O(1)$，因为dp[2][3]和newdp[2][3]是常数级空间。

快排模板

发表于 2021-08-18 分类于 算法

虽然STL库有sort()函数，但是学会快排的手撕也是很重要的。

$i = 0,j = n-1$

当$nums[i]<=nums[j]$时，j开始回退，如果出现$nums[i]>nums[j]$，交换$nums[i]与nums[j]$，然后$i++$，当$nums[i]<=nums[j]$时，$i$继续前进，如果出现$nums[i]>nums[j]$，交换$nums[i]与nums[j]$，$j$再次回退，一直执行到$j=i$为止。
再用递归将左半部分和右半部分快排即可。
