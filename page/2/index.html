11.盛最多的容器

题目描述

原题链接地址

分析

面积$s=min(h[i],h[j])*(j-i)$

双指针做法：

我们让$i,j$俩指针分别指向数组的开头与结尾，那么接下来比较两个指针指向的数字。

如果我们移动数字较大的那个指针，那么「两个指针指向的数字中较小值」$(j-i)$会减小，而「指针之间的距离」$min(h[i],h[j])$会减小，那么这个乘积$s=min(h[i],h[j])*(j-i)$会减小。因此，我们移动数字较大的那个指针是不合理的。因此，我们移动 数字较小的那个指针。

所以结论是我们移动数字较小的那个指针。

发表于 2021-08-18 更新于 2021-08-19 分类于 Leetcode

力扣每日一题（困难）

522.学生出勤记录

题目描述

原题链接地址

分析

很明显的动态规划题。

三种状态方程：

若第$i$天的出勤状态是$A$，则状态转移方程为

$dp[i][1][0]=dp[i][1][0]+\displaystyle \sum^{2}_{k=0}dp[i−1][0][k]$

若第$i$天的出勤状态是$L$，则状态转移方程为

$dp[i][j][0]=dp[i][j][0]+dp[i−1][j][k-1]$

若第$i$天的出勤状态是$P$，则状态转移方程为

$dp[i][j][0]=dp[i][j][0]+\displaystyle \sum^{2}_{k=0}dp[i−1][j][k]$

由于$i$是从第一天一直遍历到最后一天的，所以我们可以用$newdp[j][k]$来存储当天的所有状态情况，用memset(newdp,0,sizeof(newdp))来初始化该数组，然后在一天结束之后，用memcpy(dp, newdp, sizeof(dp))将$newdp$的内容拷贝到$dp$中去，从而达到省下$i$的空间。

时间复杂度为$O(n)$，因为$i$遍历了$n$天。

空间复杂度为$O(1)$，因为dp[2][3]和newdp[2][3]是常数级空间。

虽然STL库有sort()函数，但是学会快排的手撕也是很重要的。

$i = 0,j = n-1$

当$nums[i]<=nums[j]$时，j开始回退，如果出现$nums[i]>nums[j]$，交换$nums[i]与nums[j]$，然后$i++$，当$nums[i]<=nums[j]$时，$i$继续前进，如果出现$nums[i]>nums[j]$，交换$nums[i]与nums[j]$，$j$再次回退，一直执行到$j=i$为止。
再用递归将左半部分和右半部分快排即可。

HashSet

目标

不使用任何内建的哈希表库设计一个哈希集合（HashSet）。

实现 MyHashSet 类：

  • void add(key) 向哈希集合中插入值 key 。

  • bool contains(key) 返回哈希集合中是否存在这个值key 。

  • void remove(key) 将给定值key 从哈希集合中删除。如果哈希集合中没有这个值，什么也不做。

实例

分析

解决哈希表冲突的主要方法如下：

  • 链地址法：为每个哈希值维护一个链表，并将具有相同哈希值的元素都放入这一链表当中。
  • 开放地址法：当发现哈希值 $h$ 处产生冲突时，根据某种策略，从 $h$ 出发找到下一个不冲突的位置。例如，一种最简单的策略是，不断地检查$ h+1,h+2,h+3,\ldots$ 这些整数对应的位置。
  • 再哈希法：当发现哈希冲突后，使用另一个哈希函数产生一个新的地址。

我们选择链地址法，基数$base$选为$521$，正好在七夕节$/doge$。

问题描述

破译如下代换密码，要求写出对应的明文（明文用小写字母，单词间无空格和标点符号）：

解决办法

1、分析各个字母的频率，发现M的频率最高，为29，我们可以猜测M对应的明文字母为e，$d_K(M)=e$。另外，出现频率较高（频率大于10）的是B、G、K、L、T、V，我们希望这些字母对应的是t,a,o,i,n,s,h,r的一个子集的加密，但实际的频数变化很难看出它们之间的对应关系。

于是，密文变为了

2、注意形如”-M”或”M-“的两字母组，因为已经假设M解密成e。我们发现出现这种类型的最一般的两字母组是OM,MV(各出现5次),BM,MG(各出现4次),AM,LM,MK,MT(各出现3次),GM,SM,TM,VM,MB只出现1次,KM,ML则没有出现。因为MK出现了3次而KM没有出现过，所以猜测$d_K(K)=a$，此时密文变成了以下形式。

3、注意形如”M_M”的三字母组，因为MOM出现最多(3次)，所以我们可以假定$d_K(O)=v$，即$d_K(MOM)=eve$。此时密文变为了以下形式。
